# Combat / Survival Reference

Use this for arena combat, survival, wave defense, light shooters, melee brawlers, and dodge-heavy action games.

## What Must Be Clear in the Spec

- player fantasy: power, pressure, panic, mastery, mobility
- primary verbs: aim, shoot, dodge, block, kite, collect, survive
- threat cadence: constant pressure, waves, rooms, scripted encounters
- resource loop: health, ammo, stamina, cooldowns, pickups, score
- fail / win condition: death, timer, boss clear, objective complete, endless survival

## Good Clarifying Questions

- Is the main appeal aim / timing, positioning, or crowd control?
- Is the player fragile and evasive, or durable and aggressive?
- Is the game about surviving, clearing encounters, or chasing score?
- How many enemy types exist in v1?
- What keeps combat from feeling flat: movement, pickups, terrain, enemy patterns?

## Scope Traps

- too many weapons in v1
- multiple enemy archetypes before one feels good
- large maps before combat readability is proven
- upgrade trees or meta progression too early
- heavy narrative framing around a weak loop

## Healthy V1 Shape

- one arena or small set of encounter spaces
- one player attack model
- one enemy type, maybe one variant
- one clear fail state
- simple score or wave escalation

## Strong Early Slices

1. Player movement plus one attack interaction
2. One enemy / hazard that creates real pressure
3. Health / damage / defeat rules
4. Spawn or encounter pacing
5. HUD and restart flow

## Signs the Spec Is Too Thin

- enemy behavior is described only as "chases player"
- combat has no explicit risk / reward loop
- survival mode has no escalation model
- score exists but has no gameplay meaning
